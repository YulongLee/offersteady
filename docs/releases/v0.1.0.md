# OfferSteady v0.1.0 Release Notes

发布日期：待封板确认
发布阶段：早期商业化验证 / 内测候选版本

## 发布目标

v0.1.0 的目标是把当前本地原型推进到可在单台 Ubuntu 24.04 服务器上运行的验证版本。该版本用于小规模验证短信登录、资料库、面试问答、截图回答、积分页和码支付链路，不等同于完整商业化生产版本。

## Included Capabilities

- Web 应用主流程：登录入口、面试首页、资料库、面试创建、实时面试页、积分页、设备页和设置页。
- Backend API：FastAPI 基础服务、统一响应壳、健康检查和主要业务模块路由。
- 短信登录：支持阿里云个人开发者短信验证码配置，验证码发送间隔为 30 秒。
- 资料库与 RAG：支持上传简历、JD 和知识资料，并按现有后端管线做解析、索引和回答上下文接入。
- 面试回答：支持基于简历、JD 和可选知识库的回答生成流程。
- 截图回答：支持桌面助手触发截图后调用视觉模型生成回答。
- 积分和会员：支持服务端商品目录、积分余额、会员权益展示和积分页购买入口。
- 码支付：支持点击商品后由后端创建对应价格订单，并跳转码支付收银台。
- Docker 部署基线：支持 Web、Backend、PostgreSQL/pgvector、Nginx 通过 Compose 在单机启动。

## MVP Risk Classification

| Area | v0.1 状态 | 风险 | v0.1 决策 |
| --- | --- | --- | --- |
| 支付订单和会员权益 | 码支付下单、验签和入账链路已具备本地烟测 | 若账务状态仍为内存实现，后端重启会丢订单/权益 | 阻断真实付费用户；仅允许管理员/内部小额测试，直到订单、流水、权益持久化完成 |
| 用户认证会话 | 短信登录链路可用 | 如果认证仓储仍为内存，重启后 token/session 可能失效 | 内测可接受；发布说明提示重启后可能需重新登录 |
| 资料库/RAG | OSS、MinerU、Embedding、Rerank 依赖环境变量 | 第三方配置缺失会导致资料不可用或回答不引用资料 | 内测可接受；发布前需跑单项配置检查 |
| 实时语音 | 链路仍在打磨 | 桌面采集、ASR 或浏览器实时显示可能不稳定 | 不作为 v0.1 收费承诺能力；保留为实验能力 |
| 桌面助手权限 | macOS 权限和重建签名仍有摩擦 | 重装/重建后可能需要重新授权 | 内测可接受；提供人工排障说明 |
| 截图回答 | 当前主链路可用 | 模型耗时受供应商影响 | 内测可接受；不承诺 2 秒内完成 |
| HTTPS/域名 | 待服务器和域名条件确认 | HTTP 公网 IP 会影响用户信任和支付回调稳定性 | 可用于内部测试；公开用户前应配置 HTTPS 域名 |

## Paid-User Decision

v0.1.0 默认不面向真实付费用户公开售卖。码支付仅用于管理员和内部测试账号做小额链路验证。

真实付费用户开放前必须完成：

- 支付订单、积分流水、会员权益 PostgreSQL 持久化。
- 公网 `notify_url` 真实回调验收。
- 支付成功、重复回调、金额不匹配、后端重启后的回归测试。
- 明确退款、补单和客服处理流程。

## Deferred Hardening

- 完整 CI/CD、自动发布和回滚流水线。
- 蓝绿部署、灰度发布、Kubernetes 或多机高可用。
- 正式域名、HTTPS 自动续期和ICP备案流程。
- 支付对账、退款后台、审计后台和客服权限系统。
- 全量数据库迁移体系和所有内存仓储替换。
- 全链路压测、SLO、告警和成本仪表盘。
- 桌面助手 Developer ID 签名与 notarization。

## Server Acceptance Checklist

- Web 首页可通过服务器地址访问。
- Backend `/healthz` 返回 200。
- Backend `/api/v1/billing/status` 返回 200。
- 登录页面可打开，短信验证码按钮不返回 404。
- 资料库页面可打开。
- 面试创建页和实时面试页可打开。
- 积分页点击商品会打开码支付收银台。
- 码支付链接中的 `notify_url` 和 `return_url` 不包含 `127.0.0.1`。
- 若执行小额支付测试，后端日志能看到支付通知并且订单只入账一次。

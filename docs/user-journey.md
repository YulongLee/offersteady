# 用户旅程

状态：Draft

## 1. 面试前

用户可以先在“简历”“职位 JD”“知识库”三个分区维护可复用资料，再创建具体面试。进入准备页后才选择本场使用的资料；三类资料都可以为空，系统不得自动继承或授权其他资料。

面试首页对未结束会话只提供“继续面试”入口：准备中的会话进入准备页，进行中或可恢复会话直接进入实时页。准备页候选资料与资料页使用同一来源集合；确认本场清单后即可进入面试，无需重复勾选通用数据用途确认，也不需要先通过 Web 端的问题输入检查。音频和截图仍在实际启用或上传时取得针对性授权；收音、系统音频和问题检测由本地端软件检查。

空资料或部分资料不阻止开始面试，但系统必须说明个性化程度。空资料回答标记为“未使用个人资料”，不得虚构候选人的公司、项目、职责或量化结果。

关键问题：用户愿意提供多少信息？导入流程是否足够轻量？

## 2. 面试中

如果产品最终支持真实面试辅助，应明确告知音频权限、数据去向和辅助边界。产品提供信息整理与提醒，不应鼓励违反面试规则的隐蔽使用。

桌面实时页以左右两栏同时显示面试官/面试者实时对话和当前回答，用户可拖动分隔条调整宽度；平板与手机按对话、回答、紧凑操作顺序单列展示。实时页不再显示资料侧栏或现场更换资料入口。首发双声道模式只展示“面试官”和“我”：本地麦克风或耳机输入为“我”，电脑系统音频为“面试官”，不提供角色待确认或角色修改。来源无法区分时停止音频自动回答并提示使用手动提问。

系统默认只回答系统音频中已确认的完整问题；文本或边界低置信度问题可以在相关对话条目内确认或忽略，本地候选人陈述不得触发回答或用量。最新回答生成期间提供“终止回答”；成功终止不停止面试或收音，不把未完成内容作为可用建议，并释放对应积分预留。

历史答案在回答区域内通过上一条、下一条翻页；新答案到达时不打断正在阅读的历史页。手动问题输入、“回答问题”和“截图回答”组成回答栏底部的紧凑操作栏，实时按钮不展示点数。拖动分隔条、终止回答或响应式布局变化不得清空草稿、截图任务或当前答案页。

关键问题：现场最需要的是转录、提示、结构化笔记，还是情绪支持？

## 3. 面试后

用户查看摘要、问题列表、表达反馈和后续行动建议，并决定是否保留或删除本次数据。

关键问题：什么反馈最可信、最能产生下一步行动？

## 原型应覆盖的最短闭环

```text
按需维护资料 -> 创建一次面试 -> 选择资料或确认空资料 -> 进入面试会话 -> 结束 -> 查看复盘
```

第一轮原型不要求每个环节都接入真实 AI；可以使用明确标注的模拟数据验证信息架构和交互。

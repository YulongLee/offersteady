# 实时面试链路性能报告（2026-07-20）

## 结论

实时链路的本地网关、Redis 重启恢复和远程 Qwen 实时 ASR 手动提交路径已经可用，但本轮尚不能通过商业发布性能门禁：远程测试只有少量样本，现有远程剖析脚本仍使用旧 HTTP 逐帧接口，不能替代 WebSocket v2 的生产性能验证。30 分钟稳定性和并发会话仍需执行。

- macOS 本地开发环境
- 桌面端 Swift 原生采集运行时、Electron 渲染与主进程生产构建

- Redis 7.4 Compose 容器，通过 ECR Public 拉取并完成健康检查、重启恢复和 TTL 实测

## 尚未通过的门禁

1. 已完成 10 分钟协议与恢复压力测试；OpenSpec 原定 30 分钟门禁仍需在正式发布前补足。
2. 至少采集足够远程样本计算真实转录 p95，而不是以单次结果替代。
3. 在内部环境完成影子流量和按用户/会话灰度后，才可删除旧 HTTP 逐帧链路。

当前生产配置使用 `manual` 提交与 `websocket-v2`，VAD 仅作为实验对照，不是发布默认路径。

## 优化优先级

1. 增加 WebSocket v2 真实音频多样本基准，替换旧 HTTP 剖析脚本的性能结论。
2. 保持生产 `manual` 提交模式；VAD 只在提供商最终事件延迟稳定后再进入灰度候选。
3. 完成持续性测试后再决定是否关闭本地运行态回退。

# 产品知识库

`docs/` 保存跨功能、长期有效的项目事实；具体功能需求应放入 `specs/`。

- `product-vision.md`：产品定位、目标用户和成功标准。
- `user-journey.md`：面试前、面试中和面试后的体验流程。
- `architecture.md`：当前边界与未来技术架构方向。
- `privacy-and-ethics.md`：录音、转录、个人信息和辅助边界。
- `local-web-access.md`：本地网页原型的启动地址、诊断命令和巡检前提。
- `prototype-baseline.md`：当前 Web 原型受保护基线与不得静默改动的体验约束。
- `prototype-review-2026-06-30.md`：最近一轮本地原型恢复与功能审查结论。
- `commercial-material-rag.md`：商业化资料存储、OSS 路径、数据库表、RAG 会话边界与索引计费约定。

当文档与已确认 Spec 冲突时，应先澄清并统一事实来源，不允许让两者长期分叉。

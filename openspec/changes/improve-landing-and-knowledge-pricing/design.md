## Context

首页首屏已经有“AI 面试助手，助你更从容地冲刺 Offer”和免费 200 点，但页面尾部仍使用醒目的 `CLEAR BOUNDARIES` 卡片，以“不替你编造经历”和数据处理细节收尾。它适合作为信任说明，却不是帮助用户形成购买意愿的核心价值表达。

知识库当前把空集合创建与文档索引分开，空集合免费，成功建立索引按固定 20 点结算。固定费率既不能覆盖不同长度文档的成本，也和用户提出的“200 点起、每 1,000 Token 20 点”冲突。现有会员只覆盖回答与截图，知识索引全部排除；本次需要让 15 天和 30 天会员各获得两份知识材料权益。

## Goals / Non-Goals

**Goals:**

- 用直观的用户收益替代首页大面积边界说明，提高产品理解和转化意愿。
- 保留可信、安全和隐私边界，但将其放在不干扰主价值叙事的次级位置。
- 建立由服务端报价的知识材料最低消费和 Token 阶梯价格。
- 让 15/30 天会员拥有两份可审计、幂等的知识材料索引额度。
- 保持积分预留、成功结算、失败释放、目录版本和敏感内容最小日志规则。

**Non-Goals:**

- 不修改普通回答 5 点、截图回答 15 点及现有会员人民币价格。
- 不把 Token 计费扩展到实时回答、截图、语音或其他功能。
- 不承诺购买产品即可获得 Offer，也不删除隐私与真实经历边界。
- 不增加新的文件类型、突破已公布的文档大小限制或确定具体模型供应商。

## Decisions

### 1. 首页用“价值证明”替换“边界宣告”

原 `CLEAR BOUNDARIES` 大卡片替换为中文价值模块：

- 标题建议：`从听懂问题，到组织答案，现场更从容。`
- 说明建议：`结合你的简历、目标岗位和知识材料，快速抓住问题重点，生成清晰、贴合你的回答思路。`
- 三项直接收益：`实时抓住问题重点`、`回答更贴合你的经历`、`按求职节奏灵活使用`。

“AI 建议仅供参考、经历以真实情况为准、数据可管理与删除”保留为模块下方的单行信任说明，并链接隐私或使用说明。首页仍不得使用“保证拿 Offer”“标准答案”“完全准确”等无法兑现的表达。

备选方案是只润色原边界标题。它仍会让用户在页面的关键收尾位置先看到限制而不是收益，因此不采用。

### 2. 知识材料价格使用最低消费加 Token 阶梯

服务端目录发布：

```text
minimumPoints = 200
pointsPer1000Tokens = 20
pointCost = max(minimumPoints, ceil(tokenCount / 1000) * pointsPer1000Tokens)
```

因此 1–10,000 Token 均为 200 点，10,001–11,000 Token 为 220 点。无有效文本的文件不得套用最低消费直接扣费，而应返回无法索引状态。

备选方案是“200 点基础费再叠加每千 Token 20 点”。这会让最小文档从 220 点开始，与用户提出的“200 点起”不一致，因此不采用。

### 3. Token 与报价完全由服务端确定

文档上传后先进入非计费的解析与报价阶段。服务端对规范化后的可索引文本使用版本化 tokenizer 计数，并返回短期报价：

```ts
interface KnowledgeIndexQuote {
  quoteId: string;
  documentVersionId: string;
  tokenCount: number;
  billableUnits: number;
  pointCost: number;
  entitlementSource: "points" | "pass_allowance";
  catalogVersion: string;
  tokenizerVersion: string;
  expiresAtMs: number;
}
```

客户端只展示报价，不提交可信 Token 数或费用。用户确认后服务端按 `quoteId + documentVersionId` 幂等创建用量。如果规范化内容、目录或报价已过期，必须重新报价；不得静默提高已确认费用。

备选方案是在浏览器按文件大小估算。文件大小与有效文本、OCR 和 tokenizer 差异过大，也会允许客户端篡改，因此不采用。

### 4. 15/30 天会员额度代表两份成功索引的材料

3 天和 7 天会员的 `knowledgeIndexAllowance` 为 0；15 天和 30 天会员为 2。额度绑定到具体会员权益段，在该权益段开始时生效，在结束时失效。提前购买的顺延会员只有到对应权益段开始后才能使用其额度，避免重复叠加造成歧义。

一份额度覆盖一份在公开文件限制内、成功生成可用索引的文档版本，不再扣积分；失败或取消不消耗额度。相同内容和相同幂等键复用原结果，不重复消耗。替换内容形成新版本并重新索引时视为新的索引用量。额度用尽后使用积分公式报价。

这里不把额度定义为“可创建两个知识库集合”，因为空集合本来免费，无法形成有意义的会员权益。

### 5. 权益和积分共享一个索引用量状态机

索引用量保留 `quoted → reserved → processing → settled/released` 状态。积分来源在 `reserved` 时预留点数；会员额度来源只锁定一个额度，成功后消费，失败后释放。两者使用相同用量 ID 和文档版本去重，流水显示“会员知识材料额度”或具体积分，而不是伪造 0 点购买。

### 6. 计费展示集中但不隐藏规则

积分页展示公式、示例和各会员档位的知识材料权益。知识库上传确认展示本文件 Token 数、费用或剩余额度，以及操作后的预计余额。首页价格模块只需用易读摘要说明“知识材料 200 点起；15/30 天会员含 2 份”，不承载完整公式。

### 7. Token 计数和成本日志不保存正文

计费、分析和毛利日志只记录 tokenCount、点数、tokenizer/catalog 版本、文档版本 ID 和用量状态，不记录规范化文本、文件名或内容片段。原始文件和索引继续遵循现有用户删除与最小保留规则。

## Risks / Trade-offs

- [200 点最低消费会快速用完新用户赠送额度] → 上传确认明确展示影响，让用户先体验回答功能，并用真实转化和成本数据复核门槛。
- [Token 数与用户理解的字数不同] → 同时展示 Token 数、计费档位和两个示例，不把 Token 描述成字符数。
- [两份会员额度可能被超大文件滥用] → 继续执行公开文件大小、页数和正常使用限制，超限文件在报价前拒绝或要求拆分。
- [顺延会员额度生效时间难理解] → 收费页显示“随对应会员周期生效”和具体开始、结束时间。
- [边界文案弱化造成用户误解] → 首页保留一行可信说明和显著链接，登录前仍能查看隐私与使用边界。
- [旧固定 20 点规则与新规则并存] → apply 时统一修订活跃 Specs、fixtures、指南和测试，不通过客户端兼容文案掩盖冲突。

## Migration Plan

1. 为目录、报价、用量和会员权益增加版本化字段，保留旧用量的 20 点快照。
2. 发布新目录版本；新上传走 200 点最低消费与 Token 阶梯，已确认旧报价继续按原快照执行到过期。
3. 为新购买或新开始的 15/30 天权益段发放两份额度；不追溯修改已结束权益。
4. 更新积分页和知识库上传确认，再切换服务端新报价，避免用户在无说明情况下被新价格扣费。
5. 最后替换首页价值模块并同步指南、FAQ、分析事件和回归测试。

回滚时停止发布新目录，把新报价入口切回上一版本；已确认报价、已消费额度和历史流水保持不变。

## Open Questions

- 首轮用户测试需要比较“从听懂问题，到组织答案，现场更从容”与更直接的“实时回答，让面试更有底气”两种标题的点击和理解度。
- 上线前需根据真实解析、Embedding 和存储成本确认 200 点与每千 Token 20 点是否达到毛利阈值；费率仍必须可由服务端目录调整。

## Why

当前可运行 Web 原型只展示桌面伴随程序的下载与连接状态，还没有承载“准备资料—进入面试—获得回答—结束复盘”的浏览器产品闭环。需要先定义网页端的信息架构和页面职责，让 Web 成为用户实际使用面试稳的主产品，Mac 伴随程序只承担授权收音与设备状态。

## What Changes

- 新增公开首页和登录入口，清楚说明产品用途、数据边界以及 Web 与 Mac 伴随程序的分工。
- 新增登录后的应用框架，包括面试列表、准备中心、历史记录、设备与隐私设置。
- 新增面试准备页面，在同一流程中完成简历、JD、知识库和设备检查，并用就绪状态控制进入面试。
- 新增实时面试工作台，以当前问题和实时回答为视觉中心，同时提供问题历史、资料状态、截图回答、手动输入和会话控制。
- 新增面试结束复盘页，展示问题与回答记录、资料使用情况和可删除的数据，不在 MVP 中扩展为完整职业分析系统。
- 支持桌面、平板与手机浏览器；手机端作为同一 Web 会话的轻量伴随界面，不机械复制桌面三栏布局。
- 明确加载、空数据、解析失败、AI 生成失败、离线、设备未连接和权限不足等产品状态。
- 简历、JD、知识库、截图、音频转录和回答记录均按敏感数据处理；页面必须显示保存与删除入口，AI 内容必须标注为建议且不得虚构用户经历。

## Capabilities

### New Capabilities

- `web-app-shell`: 公开入口、身份入口、登录后导航、路由保护以及跨页面全局状态。
- `interview-preparation-workspace`: 创建面试并完成简历、JD、知识库、设备和隐私检查的准备流程。
- `live-web-interview-workspace`: 浏览器中的实时问题、回答、截图、历史、资料和会话控制布局与交互。
- `interview-review-and-data-controls`: 结束后的轻量复盘、会话记录管理以及敏感数据查看和删除。

### Modified Capabilities

无。当前仓库尚无已归档能力规范；本变更引用进行中的 MVP、跨设备和桌面音频提案，但不重定义它们的协议行为。

## Impact

- `apps/web` 将从单一下载页面演进为带路由的浏览器应用，并保留设备下载与连接入口。
- 前端需要统一页面外壳、设计令牌、响应式布局、页面状态模型和可访问性测试。
- API 后续需要提供面试草稿、资料状态、会话快照、问题记录、复盘摘要和删除命令；本提案保持模型、语音、解析及检索供应商可替换。
- Web 与 `apps/desktop` 通过既有配对和设备状态协议协作，桌面伴随程序不承载简历、JD 或回答正文。
- 原型测试只能使用合成或脱敏资料，并验证未授权页面不会泄露面试内容。

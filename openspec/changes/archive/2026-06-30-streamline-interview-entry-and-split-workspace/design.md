## Context

当前首页为一场准备中的面试同时展示“继续准备”和“预览工作台”，实际都围绕同一场会话，形成了没有业务价值的入口分叉。准备页虽然已经把简历、JD 和知识材料统一到 `ContextLibrarySource`，但仍有独立的“我已了解本场数据用途”复选框；设置页还提供没有真实自动删除机制支撑的保存期限下拉框。

实时页现由 `material-rail + focused-workspace` 组成，主工作区内部再把 `ConversationMonitor` 与 `AnswerWorkspace` 上下排列。此前变更刻意没有加入拖动分隔线，本次用户已经明确希望删除现场资料栏，并让对话与回答利用桌面横向空间，因此需要修订该决定。已有回答来源、资料选择 revision、说话人检测、紧凑输入栏和截图状态均继续复用。

## Goals / Non-Goals

**Goals:**

- 用一个“继续面试”入口承接准备中和进行中的会话。
- 让准备页资料候选项与资料页来源保持单一事实来源。
- 去掉没有提供额外授权价值的通用用途复选框，同时保留具体敏感操作的权限边界。
- 去掉未接入真实保存策略的期限选择器。
- 删除实时资料栏，以可拖动的左右对话/回答两栏作为桌面主布局。
- 让分隔条支持指针、键盘、响应式切换和状态保持。

**Non-Goals:**

- 不改变资料上传、解析、索引、RAG 选择 revision 或回答来源记录。
- 不允许在进行中的面试里更换资料；未来若重新引入，必须另行定义并发回答与 revision 行为。
- 不新增自动到期删除服务，也不承诺具体记录保存天数。
- 不改变收音权限、说话人识别、回答生成、截图处理、计费或复盘流程。
- 不把桌面分栏比例同步为账号级云端设置。

## Decisions

### 1. “继续面试”根据会话状态路由

首页只渲染一个 CTA。`preparing` 路由到 `/app/interviews/:id/prepare`，`live`、`paused` 或可恢复状态路由到 `/app/interviews/:id/live`；结束状态继续进入复盘，不出现在“继续面试”卡片中。按钮文案不随状态变化，卡片中的状态标签负责解释当前阶段。

备选方案是保留“继续准备”，仅删除“预览工作台”。这仍会让“准备”和“面试”成为用户需要理解的产品术语，也无法统一未来暂停恢复场景，因此不采用。

### 2. 资料页来源集合是准备选择器的唯一候选数据

准备页继续通过 `librarySources` 或未来对应 API 按 `ownerUserId`、类型和状态生成候选项，不复制来源内容。`SessionContextSelection` 只保存来源 ID 与 revision。简历和 JD 各允许零或一份，知识材料允许零到多份；处理失败、已删除或所有权不匹配的来源不可选。选择器与资料页共享状态映射和类型标签，防止两个页面对同一来源给出不同结论。

备选方案是准备页维护一个适合面试的缓存清单。它容易在删除、替换和解析状态变化时失效，也会造成用户看到“资料页没有、准备页却能选”的问题，因此不采用。

### 3. 将确认对象收敛为资料清单，权限按具体行为取得

准备页保留“确认本场资料”，因为它决定服务端检索允许清单；删除通用 `privacyConfirmed` 启动门槛和复选框。开始区显示不可勾选的简短说明，使用户在操作附近知道用途，但不制造一次勾选即可授权所有未来敏感行为的错觉。

手动模式不会申请音频权限。麦克风和系统音频由浏览器/桌面伴随程序在启用时触发明确的平台授权；截图继续先预览再上传。若后端仍保留旧 `privacyConfirmed` 字段，迁移期只读兼容，前端不得用它阻断启动，后续从协议和 fixture 删除。

备选方案是把通用复选框改成默认选中。默认选中既没有形成真实选择，也容易被误解为对音频、截图和保存的一揽子授权，因此不采用。

### 4. 设置页只展示真实行为，不提供虚假的保存期限偏好

删除“面试记录保存期限”表单控件。数据与隐私区域继续说明“原始音频默认不保存”，并链接到复盘删除或使用说明。会话记录现阶段沿用实际后端行为，用户通过复盘页执行单项或整场删除；未来只有在自动到期删除有服务端任务、状态和验证后，才重新提出期限配置能力。

备选方案是保留只读“30 天”。当前没有证据证明系统会执行 30 天删除，这会形成不真实承诺，因此不采用。

### 5. 桌面实时页采用 Grid 双栏与独立 separator

`LivePage` 删除 `materialRailOpen`、资料侧栏 DOM 和资料抽屉入口，主容器直接包含：

```text
┌─────────────────────────────┬─┬────────────────────────────────────┐
│ 实时对话                    │⋮│ 回答                     2 / 5      │
│ 面试官 / 我 / 时间 / 状态   │⋮│ 当前答案、依据、历史翻页            │
│ 待确认问题                  │⋮│ [问题输入] [回答问题] [截图回答]     │
└─────────────────────────────┴─┴────────────────────────────────────┘
```

容器使用 `grid-template-columns: minmax(conversationMin, ratio) separator minmax(answerMin, remainder)`。分隔条拖动只更新视图层 ratio，不重新挂载两栏组件。默认建议为对话 42%、回答 58%；实现时通过容器实测宽度换算百分比并夹在最小/最大范围内，避免固定像素在不同桌面失衡。

备选方案是 CSS `resize`。浏览器原生 resize 难以同时约束两栏、提供明确 separator 语义和键盘操作，因此使用受控分隔条。

### 6. 分隔条采用指针捕获与 WAI-ARIA separator 语义

分隔条使用 `role="separator"`、`aria-orientation="vertical"`、`aria-valuemin`、`aria-valuemax` 和 `aria-valuenow`，焦点时显示清晰轮廓。`pointerdown` 后调用 pointer capture，`pointermove` 依据容器边界更新 ratio，`pointerup`/`pointercancel` 清理拖动状态。左右方向键按 2% 调整，Shift+方向键按 10% 调整，Home/End 到允许边界，双击或 Enter 恢复默认比例。

比例保存为 `sessionStorage` 中按会话 ID 隔离的简单数值和布局版本；读取时校验有限数、版本和范围。它不上传服务端，也不包含资料名、问题或答案等敏感内容。

备选方案是只支持鼠标拖动。它会排除键盘和部分辅助技术用户，不符合已有可访问性要求，因此不采用。

### 7. 窄屏移除 separator，保持一份业务组件状态

桌面断点以下切换成单列，对话在上、回答与操作栏在下；separator 从可访问树中移除。React 组件不按断点渲染两套 `ConversationMonitor`、`AnswerWorkspace` 或 `CompactQuestionBar`，只由 CSS 调整同一实例布局，避免草稿、滚动和截图任务丢失。回到桌面时恢复有效 ratio。

备选方案是在平板保留可拖动双栏。有限宽度下两栏最小宽度互相冲突，拖动也会挤压触控目标，因此先以单列保证核心流程。

## Risks / Trade-offs

- [实时页不能现场换资料] → 准备页在开始前清楚展示最终清单；每条回答仍展示实际来源。未来若用户测试证明强需求，再以独立变更恢复。
- [移除通用确认被误解为减少隐私保护] → 开始区保留简短披露，音频和截图仍在具体操作时授权，自动化测试确保手动模式不启动采集。
- [拖动频繁触发流式内容重排] → ratio 只更新 CSS 变量，可用 `requestAnimationFrame` 合并高频指针事件，不改业务状态。
- [两栏最小宽度在小桌面冲突] → 容器低于可行宽度时直接使用单列断点，不生成不可用的窄栏。
- [移除期限选择器后用户不清楚记录保存多久] → 只陈述真实策略并保留删除入口；在服务端策略确定前不承诺虚假期限。
- [旧活跃规范仍要求资料栏、上下布局和用途确认] → apply 的第一阶段先修订相关规范和文档，再改代码与测试，避免实现与 Spec 分叉。

## Migration Plan

1. 修订相关活跃 OpenSpec：删除并行入口、通用用途勾选、保存期限偏好、实时资料栏和上下桌面布局要求。
2. 让首页“继续面试”基于会话状态路由，并更新导航与恢复测试。
3. 统一资料页与准备页的来源派生和状态映射，移除 `privacyConfirmed` 启动门槛与 UI。
4. 删除设置页保存期限控件，确认复盘删除和原始音频默认策略仍可见。
5. 删除实时资料栏及相关视图状态，重构为同一组件实例的桌面双栏/窄屏单列布局。
6. 增加 separator 指针、键盘、边界、状态保存和响应式测试，并在手机、平板和桌面进行浏览器验证。
7. 更新用户旅程和使用说明，运行 Web 测试、类型检查、生产构建及相关 OpenSpec 严格校验。

回滚时可以临时恢复固定 42/58 双栏或单列布局，但不得恢复两个首页入口、虚假保存期限或无法解释的资料候选副本。

## Open Questions

- 默认 42/58 与两栏最小宽度需要在实际内容和 1200–1440px 桌面原型中验证后最终确定。
- 进行中的面试是否永远禁止换资料，需在本轮简化交互测试后决定；本变更明确按“只在开始前选择”实现。

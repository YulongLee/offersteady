## Why

实时收音只有在系统能够区分“面试官正在提问”和“候选人正在回答”时，才能可靠生成建议。首发版本已经拥有独立的本地麦克风与系统音频来源，因此角色应由可信声道直接确定，避免把内部推断和“角色待确认”暴露给面试中的用户。

## What Changes

- 对麦克风与系统音频分别进行语音活动检测、回声/重复段抑制和实时转录；匿名 speaker 标签只用于会话内去重与排序。
- 将本地麦克风或耳机来源固定映射为 `candidate`，系统音频固定映射为 `interviewer`。
- 支持系统音频中一名或多名远端参与者，但 Web 只展示“我”和“面试官”两个角色。
- 移除用户可见的 `unknown`、角色置信度与角色纠正；来源异常时降级到手动提问而不是猜测第三角色。
- 只有高置信度面试官话语、达到问题完整性条件并经过短暂端点确认后，才自动创建回答任务。
- 候选人回答、寒暄、确认词、未完成句子、低置信度内容和双方重叠语音不得自动触发可靠回答。
- 允许用户确认内容不清晰的问题或手动提交问题，但不要求确认说话人角色。
- 来源、转录或问题边界发生修订时使用稳定事件 ID 和版本，避免流式转录重复触发同一问题。
- 当双声道不可用或只剩混合音频时，停止音频自动回答并保留手动提问路径。
- 默认不建立跨会话永久声纹，不把说话人嵌入用于身份识别；原始音频仍按既有策略短暂处理并默认不保存。
- 增加噪声、回声、打断、多人、口音、中文/英文混合和角色误判的评测集。

## Capabilities

### New Capabilities

- `speaker-aware-question-detection`: 从双声道来源固定映射面试角色、识别完整面试官问题并安全触发回答。

### Modified Capabilities

无。相关桌面音频和实时回答能力仍处于未归档变更中，本变更以独立能力补充角色感知的问题检测。

## Impact

- `packages/protocol` 需要增加带来源、固定展示角色、时间范围、修订版本和触发状态的转录事件，并兼容隔离旧 `unknown` 事件。
- 桌面伴随程序继续发送独立麦克风与系统音频，不生成回答。
- 服务端需要新增可替换的语音活动、转录、回声抑制、问题分段和触发编排模块。
- `apps/web` 只显示“我/面试官”实时转录、来源降级和问题触发状态。
- 问题和回答计费只能在去重后的问题触发确认后发生，候选人语音或重复转录不得产生用量。
- 说话人特征和音频属于敏感数据；必须最小化处理、限制第三方传输、禁止普通日志记录并默认不跨会话保留。

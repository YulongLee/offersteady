## Context

桌面伴随程序把用户选定的本地麦克风或耳机和电脑系统音频作为独立来源传输。首发版本据此只需要两个用户角色：本地输入是“我”，系统音频是“面试官”。现有三态角色推断会把 `unknown`、置信度和纠正操作暴露到面试现场，增加干扰。

固定展示角色不代表任何系统音频都可触发回答。回声、重复转录、重叠、未完成句子和非问题反馈仍必须在回答触发前被过滤。

## Goals / Non-Goals

**Goals:**

- 用可信来源稳定映射“我/面试官”两个展示角色。
- 只让最终、完整、唯一的系统音频问题自动触发回答。
- 在来源缺失或混合时安全降级到手动输入，不猜测第三角色。
- 保留事件修订去重、回声抑制、隐私最小化和合成评测。

**Non-Goals:**

- 不识别远端参与者现实身份、姓名、情绪或人口属性。
- 不在首发版本对单个混合音轨做自动角色推断。
- 不把匿名 speaker ID 展示成“面试官 1/2”或新的用户角色。
- 不绑定单一转录、问题检测或模型供应商。

## Decisions

### 1. 来源先于展示角色

```text
microphone / headset -> candidate -> 我
system audio         -> interviewer -> 面试官
```

协议保留 `sourceId`、`sourceKind`、时间范围、revision 与可选匿名 speaker ID。匿名 ID 仅供去重、排序和转录修订，不参与用户角色判断。备选的语言模型或声纹猜测更慢、更难解释，也会引入“角色待确认”，因此不采用。

### 2. 回声过滤先于系统音频问题触发

```text
独立声道 -> VAD -> 跨声道回声/重复抑制 -> 转录修订合并
         -> system 来源 -> 端点 + 完整性 + 问题意图
         -> QuestionConfirmed
```

麦克风来源永不自动触发回答。候选人的声音若回灌到系统音频，先通过时间窗和相似度抑制。无法安全判断的冲突片段不触发回答，但已有会话和手动提问保持可用。

### 3. 来源异常不进入第三角色

只有混合音频、来源断开或旧客户端发送 `unknown` 时，系统发布来源降级状态。Web 在对话列表外提示检查桌面程序或使用手动提问，不把不可靠文本强制显示为“我”“面试官”或“角色待确认”。

兼容层可在一个协议版本内解析旧 `unknown`，但新 fixture、服务和客户端不得继续创建该用户可见状态。

### 4. 问题质量与角色分开判断

角色由来源确定，问题是否可回答仍由最终转录、端点、完整性、意图和去重共同决定。低文本置信度或重叠片段可提示“问题内容不清晰”，允许用户手动输入或编辑，但不出现角色纠正。

### 5. 唯一确认事件控制回答与计费

只有唯一 `QuestionConfirmed` 事件可创建回答任务和积分预留。麦克风内容、回声副本、非问题反馈、同一 revision 重放和来源降级事件不得产生用量。

### 6. 会话内数据最小化

原始音频仍按既有短暂处理策略默认不保存。用于回声匹配或匿名 speaker 去重的临时特征只存在于本场内存或短期加密缓存，会话结束后清除；日志不得记录原始音频、完整敏感转录或跨会话 speaker 身份。

### 7. 评测以误触发和来源路由为核心

合成或授权评测覆盖双声道、多个远端说话人、回声、噪声、打断、重叠、中英文、来源断开和混合音频。门槛至少包含系统音频问题召回、麦克风误触发、重复触发、来源路由与端到端延迟。未达阈值时关闭自动回答并保留手动模式。

## Risks / Trade-offs

- [系统音频包含候选人回声] → 在问题触发前执行跨声道回声和重复抑制。
- [共享设备破坏双声道假设] → 显示来源降级并切换手动提问，不猜测第三角色。
- [多个远端参与者无法区分] → 内部可保留匿名 ID，用户界面统一显示“面试官”。
- [严格降级会漏掉问题] → 保持实时状态提示和手动问题输入；漏答风险低于循环误答和错误扣费。
- [供应商能力不一致] → 转录、回声抑制和问题检测均使用可替换适配器。

## Migration Plan

1. 更新协议，使新事件必须携带可信 `microphone | system` 来源并兼容隔离旧 `unknown`。
2. 将服务端支持路径改为固定来源路由，保留回声、修订和问题检测。
3. 更新 Web 为两角色展示与来源降级提示，删除角色确认和纠正。
4. 使用合成双声道 fixture 运行服务、Web、协议和 AI eval。
5. 若误触发指标不达标，关闭音频自动回答并保留手动模式。

## Open Questions

- 无阻塞问题。混合音频自动角色推断不属于首发范围，后续如需要必须单独提案。

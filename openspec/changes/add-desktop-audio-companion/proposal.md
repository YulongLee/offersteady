## Why

OfferSteady 的产品界面应当通过同一个响应式 Web 应用在电脑和手机上访问，但浏览器对系统音频、后台采集和平台级权限的支持有限。需要增加一个可下载安装的轻量桌面伴随程序，只负责在用户明确授权和可见控制下采集面试音频，并把处理结果桥接到同一 Web 会话。

## What Changes

- 保持电脑浏览器和手机浏览器使用同一个 Web 产品入口，无需分别维护两套业务界面。
- 在电脑 Web 页面提供与操作系统匹配的桌面伴随程序下载和安装引导。
- 桌面伴随程序支持选择麦克风、系统音频或受支持的组合输入，并持续展示音量与采集状态。
- 桌面伴随程序通过短期授权与当前 Web 面试会话绑定，将音频流或转录片段安全发送到服务端。
- Web 与手机页面同步展示桌面伴随程序的在线、授权、采集中、暂停、错误和断开状态。
- 每次采集必须由用户显式启动，并提供随时暂停、停止、撤销设备和删除会话数据的入口。
- 首版不提供隐蔽模式、无状态指示的后台监听或绕过操作系统权限。

## Capabilities

### New Capabilities

- `desktop-audio-companion`: 定义桌面伴随程序的下载、安装、权限、音频源选择、可见采集控制和本地状态。
- `audio-session-bridge`: 定义桌面程序与 Web 面试会话的授权绑定、音频/转录传输、状态同步、重连和撤销。

### Modified Capabilities

无。该能力依赖尚未归档的 `define-interview-assistant-mvp` 和 `add-cross-device-interview-workspace`，不修改已归档能力规范。

## Impact

- Web 页面新增桌面程序下载、设备绑定、权限诊断和采集状态组件。
- 新增可安装的 macOS 桌面伴随程序，首发支持 Intel x64 与 Apple Silicon arm64，并保留后续 Windows 适配边界。
- 服务端新增短期设备凭证、音频接入或转录事件入口、连接状态和会话撤销能力。
- 涉及麦克风、系统音频和第三方语音数据处理，需要明确授权、传输加密、最小保存和审计边界。
- 构建与发布流程需要处理桌面应用签名、安装包、版本检查和平台兼容测试。

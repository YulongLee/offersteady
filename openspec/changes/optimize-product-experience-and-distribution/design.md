## Context

公开首页、身份入口、资料库、收费页和下载页已经有可操作原型，但它们分别沿用了“Mac 首发”“本机原型身份”“固定晚上问候”“资料只读”和低价测试目录等阶段性假设。新的体验横跨 Web、API、协议、桌面构建与外部微信身份服务，且会影响在途的 Web 应用、积分会员、每场资料选择和桌面伴随程序变更。

目标用户需要先理解产品能解决什么问题，再决定登录、准备资料和安装伴随程序。价格与扣点必须在操作前可预测；平台兼容性必须以实际发布的签名安装包和运行时能力为准。

## Goals / Non-Goals

**Goals:**

- 用平台中立、结果导向的首页表达产品价值与差异。
- 提供安全的微信登录/注册与已有账号绑定路径。
- 让首页文案适配用户时区与当前任务，避免固定时段误判。
- 完成资料集合和资料文档的增删、解析、索引、失效与计费闭环。
- 建立以 3 天 ¥69.90 为锚点的会员阶梯和与单位成本相匹配的积分包。
- 提供 macOS arm64、macOS x64、Windows x64 三种可信下载入口。

**Non-Goals:**

- 本变更不接入真实微信或支付宝支付回调，继续沿用已定义的人工核验边界。
- 本变更不承诺 Windows ARM64、Linux 或移动端原生伴随程序。
- 本变更不改变回答内容的真实性边界，也不允许隐藏录音或规避面试规则。
- 本变更不在客户端实现微信密钥交换、价格授权、积分发放或安装包签名。

## Decisions

### 1. 首页采用“问题—能力—证据—行动”信息架构

首屏主张使用“结合你的简历、目标岗位和个人资料，实时整理有依据的回答思路”，下方依次展示个性化上下文、语音/手动/截图输入、具体来源、跨设备协同和数据控制。平台名称只出现在下载与兼容性区域，不再用“网页负责什么、Mac 负责什么”作为核心卖点。

选择这一结构是因为用户首先购买的是面试辅助价值，不是客户端架构。备选方案是继续突出 Mac 首发，但会让 Windows 用户误认为产品永远不可用，也占用首屏解释内部实现。

公开文案使用受版本控制的内容配置和功能开关。尚未发布的 Windows 能力只能显示“即将支持”或真实可用范围，不得先行承诺。

### 2. 登录后首页使用任务状态优先、时间问候次之

首页主标题优先根据当前任务生成：有未结束面试时显示“继续这场面试”，没有进行中任务时显示“准备好下一场面试了吗？”。卡片只提供“继续面试”，再根据会话状态进入准备或实时页面。问候作为较弱的辅助文本，按浏览器 IANA 时区计算早上、中午、下午、晚上和深夜；时区缺失、服务端渲染不一致或时间不可用时统一回退为“你好”。

备选方案是只使用动态时段问候，但它仍然没有告诉用户下一步做什么，而且跨时区渲染容易闪烁。

### 3. 微信身份只通过服务端授权码交换

Web 端发起微信授权或展示服务端生成的短期二维码，回调只携带一次性 `state` 和授权码。API 端完成密钥交换，保存提供方主体标识的加密映射，不向客户端暴露 `AppSecret`、长期令牌或开放平台内部字段。

首次成功授权创建 OfferSteady 账号；已登录用户可显式绑定。若微信身份已属于另一账号，系统阻止静默合并并进入经过重新验证的账号合并流程。用户必须至少保留一种可恢复登录方式后才能解绑。

备选方案是把微信昵称或手机号当作唯一账号键，但这些字段可变化、可缺失或不唯一，不能安全完成身份关联。

### 4. 资料库拆为集合、文档和索引作业

`KnowledgeCollection` 只保存名称、所有者和生命周期；`KnowledgeDocument` 保存文件元数据、内容摘要、解析状态和版本；`IndexJob` 使用幂等键关联文档版本、预计点数、预留和结算状态。创建空集合免费，真正产生解析、嵌入和存储成本的索引作业才计费。

MVP 默认支持 PDF、DOCX、TXT 和 Markdown，并在上传前显示大小、页数/字符数上限。服务端解析有效文本后按 `max(200, ceil(Token / 1000) × 20)` 返回版本化报价；用户确认后再创建任务。失败、取消、未变化内容的重复上传和同一幂等键重放释放预留，不结算。

删除采用立即停止未来检索的软删除与异步内容清理。每场面试选择不会被系统偷偷替换；历史回答只保留来源名称、版本和删除标记，不保留被删除内容。

备选方案是“创建集合即扣点”，实现简单但与实际成本脱节，空操作收费会明显损害信任。

### 5. 价格由服务端目录版本控制并以毛利门槛发布

首发建议目录：

| 商品 | 价格 | 相对 3 天日均价 |
|---|---:|---:|
| 3 天会员 | ¥69.90 | 基准，¥23.30/天 |
| 7 天会员 | ¥129.90 | 约便宜 20% |
| 15 天会员 | ¥219.90 | 约便宜 37% |
| 30 天会员 | ¥329.90 | 约便宜 53% |
| 300 点 | ¥39.90 | ¥0.133/点 |
| 800 点 | ¥89.90 | ¥0.112/点 |
| 2000 点 | ¥199.90 | ¥0.100/点 |

会员覆盖正常个人使用范围内的普通回答和截图回答；3/7 天会员的资料解析仍扣点，15/30 天会员每个生效权益段包含 2 份知识材料索引额度，超出后按积分报价。新用户 200 测试点、普通回答 5 点和截图回答 15 点保持不变；知识材料最低 200 点、每 1,000 Token 20 点，仍需通过单位经济门槛发布。

目录发布必须计算保守成本与目标毛利。价格调整生成新目录版本，已创建订单保留商品快照，已购买权益不被追溯修改。

一次性积分兑换码不属于商品目录，也不覆盖费率。兑换成功只向现有钱包追加服务端定义的 `redemption_credit`，目录版本、积分包价格、会员价格和毛利校验保持不变。

### 6. 下载页由签名分发清单驱动

API 返回受控 `ReleaseManifest`，至少包含平台、架构、版本、最低系统版本、文件大小、下载地址、SHA-256、签名/公证状态、发布时间和能力标记。Web 根据浏览器平台做推荐，但始终允许手动选择：

- macOS Apple Silicon（arm64）
- macOS Intel（x64）
- Windows 10/11（x64）

macOS 包必须完成 Developer ID 签名与公证；Windows 包必须完成代码签名。未签名、校验值缺失、已撤回或尚未构建的版本不得显示为可下载。运行时握手继续报告实际麦克风、系统音频和协议能力，下载成功不等于所有能力都可用。

备选方案是继续只发布 macOS universal 包。它减少一个安装包，但用户无法直观看懂芯片选择，也不满足 Windows 分发目标；因此改为三张明确下载卡，并可在构建层继续复用同一代码库。

## Risks / Trade-offs

- [微信网页授权需要主体资质且不同入口能力不同] → 用身份适配器隔离开放平台与公众号模式，资质确认前保留原型入口并明确标识。
- [提高价格降低初次转化] → 保留 200 点试用，展示日均价与适用人群，用实际转化和毛利数据做版本化调整。
- [“不限次会员”被滥用] → 明示正常个人使用、单账号并发与安全限流；资料索引排除在会员范围外，不使用隐藏扣点。
- [资料索引报价与实际成本偏差] → 服务端按文件规模预估并设置保守上限，记录无内容的成本指标，低毛利目录不得发布。
- [删除资料破坏历史复盘] → 立即阻止未来检索，仅保留最小来源墓碑；原内容按删除流程清理。
- [Windows 系统音频能力与 macOS 不一致] → 下载清单和运行时能力共同驱动 UI，不用统一文案掩盖平台差异。
- [错误芯片包导致安装失败] → 自动推荐加人工选择，显示“如何查看芯片”，安装帮助覆盖误装恢复。

## Migration Plan

1. 增加新首页文案与动态入口规则，在功能开关下进行响应式和可访问性验证。
2. 建立微信身份适配器和合成回调测试；取得资质前不启用真实提供方。
3. 迁移资料库合成数据到集合/文档模型，先启用 CRUD，再启用索引预留与结算。
4. 发布新目录版本；旧订单与权益继续读取原商品快照，禁止前端硬编码价格。
5. 建立三平台发布清单和签名流水线，只有验证通过的构建才进入下载页。
6. 逐步启用 Windows 能力，先完成安装、登录、连接与手动输入，再以测试结果开放系统音频。

回滚时可关闭微信入口、资料索引收费、新价格目录或某个平台下载项；不能回滚已确认订单、已结算流水或已授予权益。

## Open Questions

- 首发微信登录采用开放平台网站应用扫码，还是公众号内 OAuth？取决于主体资质与主要获客入口。
- Windows 首发最低版本最终定为 Windows 10 22H2 还是仅 Windows 11？
- 资料超过 5 MB/50 页后的阶梯报价，需要在获得解析与嵌入成本样本后确定。
- 会员价格是否需要首发优惠券，而不是直接降低公开标价？

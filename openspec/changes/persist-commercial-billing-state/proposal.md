# Persist Commercial Billing State

## Why

当前官方订单、购买积分、欢迎积分、会员权益和知识库计费预留保存在后端进程内存。服务重启或多实例切换后，用户余额、订单和权益可能消失，支付回调也无法可靠幂等，不满足商业化收费要求。

## What Changes

- 使用 PostgreSQL 持久化钱包流水、官方订单、会员权益、知识库报价和预留。
- 支付成功、积分入账或会员发放在同一数据库事务内完成，并支持重复回调幂等。
- 欢迎积分按用户全局只发放一次。
- 生产环境数据库不可用时账务能力失败关闭，不回退到进程内存。
- 开发和测试环境保留内存实现，保持现有开发范式和 API 契约。

## Non-goals

- 不修改商品目录、售价、支付签名算法或页面原型。
- 不在本批次实现退款、运营后台、发票或对账文件导入。
- 不迁移历史进程内存数据；已有持久化兑换流水继续保留并纳入统一余额。

## Capabilities

- `commercial-billing-persistence`: 可重启、并发安全且幂等的商业账务状态。

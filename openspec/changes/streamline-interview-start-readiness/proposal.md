## Why

当前准备页把“进入面试”和“问题输入方式检查”绑在一起，导致用户创建面试、确认资料后仍可能被 Web 端拦住，流程上显得多余。产品逻辑应更直接：Web 负责确认本场资料并进入面试，音频来源、问题识别和收音诊断由已下载的本地端软件或实时页具体输入操作负责检查。

## What Changes

- 调整准备页开始条件：用户创建面试并确认本场资料清单后即可进入实时面试页，资料清单可以为空或部分选择。
- 移除 Web 准备页中“问题输入方式必须就绪”作为开始面试的阻塞项，不再因为未连接本地端、未授权音频或未选择手动模式而禁用“开始面试”。
- 保留输入能力说明，但将其改为非阻塞提示：音频/问题检测由本地端软件检查；手动输入和截图回答在实时页始终作为兜底入口。
- 本地端软件承担开始前或连接后的收音、系统音频、麦克风和问题检测可用性检查，并把诊断状态同步给 Web 展示。
- 不改变已批准的实时页布局，不移动手动输入、回答问题、截图回答、终止回答等操作。

## Capabilities

### New Capabilities

- `desktop-input-readiness-diagnostics`: 定义下载的本地端软件负责音频来源、收音能力和问题检测检查，并将状态同步给 Web，而不是由 Web 准备页阻塞进入面试。

### Modified Capabilities

- `streamlined-interview-entry`: 将开始面试的必要条件从“资料清单确认 + 至少一种问题输入方式可用”调整为“资料清单确认”；问题输入能力检查变为非阻塞提示和实时页/本地端责任。

## Impact

- Affected frontend: `apps/web/src/App.tsx` 的准备页检查文案、完成度、开始按钮启用逻辑及相关测试。
- Affected backend/session: 如果后端 session start 接口存在输入方式 gate，需要同步放宽为只依赖资料确认与会话权限。
- Affected specs/docs: `openspec/specs/streamlined-interview-entry/spec.md`、用户旅程和使用说明中的准备流程描述。
- Privacy impact: 本变更不扩大音频、截图或个人资料授权范围；进入面试页不代表自动开始收音。音频权限和采集仍必须在本地端软件或用户触发对应能力时明确处理。

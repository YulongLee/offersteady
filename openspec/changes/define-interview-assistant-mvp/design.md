## Context

OfferSteady 当前只有产品文档和手工 Spec 草案，没有可运行应用。MVP 需要把简历、JD、用户知识库、实时问题和截图问题接入同一回答编排流程，同时避免在产品方向尚未验证时绑定具体模型、语音或向量数据库。

主要约束如下：

- 产品仍处于原型验证阶段，应优先验证核心闭环而不是生产规模。
- 简历、JD、音频、截图和知识材料都可能包含敏感信息。
- 实时辅助需要清楚展示输入、处理状态、输出来源和失败降级。
- 首版必须允许替换模型、解析器、语音转录和检索实现。

## Goals / Non-Goals

**Goals:**

- 建立从“上传上下文”到“开始面试”再到“实时或截图回答”的完整 MVP。
- 复用统一的上下文检索与回答生成管线。
- 让用户能够检查、修改和删除用于回答的资料。
- 用清晰的模块边界保留后续升级桌面端、实时语音和生产级 RAG 的空间。
- 为每项 AI 能力建立可测试的输入、输出和失败状态。

**Non-Goals:**

- 首版不实现企业招聘管理、面试官协作或支付系统。
- 首版不承诺系统音频捕获、隐蔽录音或后台持续监听。
- 首版不自动代表用户发送答案，也不保证 AI 建议完全正确。
- 首版不绑定特定模型、向量数据库或云平台。
- 首版不以长期保存原始音频和截图为默认行为。

## Decisions

### 1. 采用桌面优先的 Web MVP

首版使用适合桌面面试场景的响应式 Web 工作区，核心布局为左侧资料状态、顶部操作区和中央实时回答区域。Web 方案最适合快速验证上传、会话和截图流程；如果后续确认必须捕获系统音频或提供悬浮窗，再增加桌面客户端。

备选方案是直接构建桌面应用，但这会在核心价值尚未验证时增加平台权限、签名和分发成本。

### 2. 使用模块化单体和适配器边界

MVP 由一个客户端和一个可信 API 服务组成，服务端按资料解析、检索、面试会话、截图理解和回答生成划分模块。外部模型、转录和检索能力通过适配器接入，业务层只依赖统一接口。

相比一开始拆分微服务，模块化单体更容易开发和调试，同时保留未来拆分高负载实时链路的可能。

### 3. 建立统一的上下文模型

简历、JD 和知识库材料解析为带来源标识的文本片段。回答请求只传递当前问题、会话状态和检索到的最小必要片段，输出携带使用了哪些上下文类型的元数据。

简历与 JD 是开始上下文化面试的必要资料；知识库为可选增强。用户仍可在资料不足时进入受限的演示模式，但界面必须说明回答不会完全个性化。

### 4. 统一实时与截图回答的生成管线

实时文本/转录和截图识别最终都转换为规范化问题对象，再进入：问题分类、上下文检索、回答生成、结构化展示。截图路径额外保留 OCR/视觉识别结果和题型信息。

统一管线可以减少 Prompt 和业务规则漂移；不同题型通过回答策略而不是复制完整流程实现。

### 5. 采用显式状态机管理会话

面试会话至少包含 `preparing`、`ready`、`active`、`paused`、`ended` 和 `error` 状态。只有明确的用户操作才能开始或结束采集，界面持续展示当前状态，并提供手动输入作为语音链路失败时的降级路径。

### 6. 数据默认最小化保存

原始音频和截图默认只在完成当前请求所需的时间内保留；持久化简历、JD、知识库和会话记录必须可由用户查看和删除。日志不得包含原始文档内容、完整回答上下文或认证凭据。

## Risks / Trade-offs

- [Web 端无法满足系统音频或悬浮窗需求] → MVP 先验证手动输入、授权麦克风与截图流程，确认需求后再评估桌面端。
- [多个上下文来源造成错误检索或回答污染] → 为片段保留来源、优先级和版本，并在回答界面显示使用的上下文类型。
- [实时回答延迟影响使用体验] → 流式展示处理状态和部分结果，限制单次检索上下文，并记录分阶段延迟。
- [截图或文档包含敏感数据] → 默认短期处理、限制日志、提供删除能力，并在上传前说明用途。
- [AI 输出被误认为标准答案] → 明确标记为建议，展示不确定性，并允许用户查看回答依据。
- [MVP 同时覆盖五项能力导致范围过大] → 按资料上下文、统一回答管线、实时入口和截图入口分阶段交付，每一阶段都可单独演示。

## Migration Plan

1. 保留现有 `docs/` 作为长期产品背景，将本变更作为 OpenSpec 下的首个正式功能提案。
2. 先建立应用骨架和合成数据演示，不连接真实用户资料。
3. 依次接入简历/JD、知识库、实时问题和截图问题。
4. 完成验收与隐私检查后，将能力规范归档到 `openspec/specs/`。
5. 如果实施失败，可移除新应用代码和未归档变更，不影响现有产品文档。

## Open Questions

- 首版面试场景是真实面试辅助、模拟面试，还是同时支持两者？
- 实时输入首版必须支持麦克风转录，还是先支持手动粘贴问题？
- 是否要求捕获系统音频或以悬浮窗覆盖会议软件？
- 简历、JD 和会话记录的默认保存期限分别是多少？
- 截图回答是否需要支持代码执行，还是只提供分析和回答建议？

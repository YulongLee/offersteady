## Overview

这次变更只解决一件核心事情：让实时回答区更像“可直接使用的答案”，而不是“需要再读一层 UI 结构才理解的建议卡片”。具体做法是把回答策略彻底下沉到后端 prompt 体系，前端只负责把真实回答结果清楚地展示出来。

## Goals

- 回答区优先显示模型返回的正文
- 让 prompt strategy 在后端统一维护
- 保留必要的状态、来源和历史能力
- 不改变当前页面主布局和交互入口

## Non-Goals

- 不新增新的回答模式选择器
- 不修改快答、截图回答、开始面试或最近面试逻辑
- 不在这次变更中扩展新的模型供应商或新的 prompt 存储系统

## Design Decisions

### 1. 前端回答区从“建议卡片”改为“正文优先”

当前 `AnswerWorkspace` 默认会渲染：

- 回答标题
- outline 提纲
- 展开详情
- 模型推断
- 来源块

这套结构更像原型展示，不像真实现场。新的原则是：

- 回答正文放在最显眼位置
- 当前问题、回答状态、历史翻页继续保留
- 来源信息缩成次级信息
- `inference`、固定 outline 等不再作为主展示层

### 2. 回答策略后移到 Chat Service 与 prompt 模板

回答应该是“最终可说出口的话”，而不是前端再根据 `outline/detail` 二次拼理解。因此后端负责：

- 决定回答是简洁版、正文版、STAR 风格还是分点版
- 在 prompt 中约束真实性、基于资料、不要虚构
- 返回适合前端直接展示的正文文本

前端不再承担这些策略。

### 3. 数据契约保持兼容过渡，但前端不再依赖复杂字段

为了降低改动风险，这次实现允许后端暂时继续返回已有结构字段，但前端回答区改成：

- 优先使用 `detail` / 主文本字段作为正文
- 不再把 `outline` 渲染成主要区块
- 不再把 `inference` 渲染为固定说明块

后续如要进一步收缩 API 契约，可以再单独起 change。

## Frontend Changes

- 重构 `AnswerWorkspace` 的完成态展示
- 将正文作为主要排版内容
- 保留失败、生成中、取消、历史翻页、终止回答等状态
- 将来源和版本信息压缩到回答底部或标题辅助区

## Backend Changes

- 检查 Chat Service 当前 prompt 是否仍偏向“先提纲、后说明”
- 将真实使用时的回答策略写入 `ai/prompts/chat-service/`
- 保证 live-answer 返回内容可被前端直接作为正文展示

## Risks and Mitigations

- 风险：回答区信息过度简化后，用户不知道依据来自哪里
  - 缓解：保留最小化来源与版本信息，但不再抢占主内容

- 风险：前端去掉 outline 后，已有测试和 fixture 不匹配
  - 缓解：先改展示优先级与测试断言，不强制一次性删掉所有旧字段

- 风险：prompt 变化导致回答风格不稳定
  - 缓解：把策略集中在后端 prompt 文件，并补充 eval 覆盖

## Verification

- 回答区完成态回归测试：正文优先显示
- 历史回答回归测试：正文仍可翻页查看
- 失败和生成中状态测试：不被正文化改动破坏
- `ai/prompts/chat-service/` 与 `ai/evals/` 的对应更新

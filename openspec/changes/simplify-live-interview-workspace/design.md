## Context

当前原型使用左资料栏、中回答区、右历史/转录/设备栏的三栏结构。上一个变更把手动输入和截图入口移动到了回答主区，但 `LiveActionComposer` 仍是较大的卡片，右栏仍长期占用宽度，回答区还限制在较窄的最大宽度。真实面试中，用户最需要持续看到双方实时对话和当前答案，其他信息应围绕这两件事服务。

本变更只重构 Web 实时工作台的信息架构和交互，不改变音频采集、说话人检测、回答生成、截图识别或服务端计费算法。

## Goals / Non-Goals

**Goals:**

- 移除左侧资料栏，把宽度全部交给实时对话和回答。
- 删除永久右栏，把转录、历史和设备状态放到更符合任务的位置。
- 用实时对话与答案两个区域表达主工作流，并确保只回答面试官问题。
- 在答案区域内部提供稳定、可理解的历史翻页。
- 将手动操作压缩为输入框、“回答问题”和“截图回答”两个小按钮。
- 从实时操作区移除点数文案，同时保留服务端计费安全与积分页说明。

**Non-Goals:**

- 不改变自动问题检测阈值、模型 Prompt、RAG 或截图识别流程。
- 不增加全文搜索、历史答案批量管理或多窗口模式。
- 不把实时对话扩展为长期保存的会议转录产品。
- 不删除暂停、恢复、结束和错误恢复能力；资料调整限定在开始前准备流程。

## Decisions

### 1. 桌面布局使用无资料侧栏的双栏工作区

桌面网格只保留实时对话与回答两列，中间设置可访问分隔条。实时页不展示资料侧栏、折叠按钮或现场调整入口；每条回答本身继续展示实际来源。分栏比例以会话 ID 保存到 `sessionStorage`，不会成为跨账号永久偏好。

右侧栏完全移除：设备连接压缩到顶部状态条；转录迁移到实时对话区；历史迁移到答案翻页。这样不是丢失功能，而是取消永久空间占用。

备选方案是保留可折叠资料栏。资料已在开始前确认，现场常驻或抽屉入口仍占用注意力，因此不保留。

### 2. 主区固定为可调整的实时对话与答案两栏

桌面主区使用左右结构：左侧 `ConversationMonitor` 展示近期对话，右侧 `AnswerWorkspace` 展示答案与翻页，并在底部放置 `CompactQuestionBar`。默认比例为 42%/58%，用户可通过指针或键盘调整；窄屏按内容自然堆叠且不显示分隔条。

```text
┌──────────────────────────┬─┬──────────────────────────┐
│ 实时对话                 │⋮│ 回答              2 / 5 │
│ 10:31 面试官：项目难点？ │⋮│ 当前问题与回答建议       │
│ 10:32 我：项目背景是……   │⋮│ [上一条] [下一条]        │
│                          │⋮│ [问题] [回答] [截图回答]  │
└──────────────────────────┴─┴──────────────────────────┘
```

### 3. 实时对话组件接管双角色转录与不清晰问题

`SpeakerPanel` 重构为 `ConversationMonitor`，按 `startedAtMs` 排序并用稳定 segment ID 更新 revision。每条显示时间、角色和最终/临时状态。自动滚动只在用户位于底部附近时发生，避免阅读较早内容时被强行拉回。

文本或边界低置信度问题的“确认问题 / 忽略”放在对应对话片段内。角色由来源固定；本地麦克风片段只显示，不进入回答管线。只有服务端确认的系统音频问题或用户显式确认事件才创建答案，并继续使用稳定问题 ID 去重。

备选方案是把确认操作留在输入栏。确认操作与具体转录片段关系更强，放在对话中更容易理解，也让输入栏保持紧凑。

### 4. 答案历史使用稳定 ID 翻页而不是右侧列表

回答状态增加 `viewingAnswerId: string | null`：`null` 表示跟随最新答案，非空表示固定查看该答案。翻页顺序来自不可变问题/答案 ID 列表，而不是保存数组索引；新答案插入头部时不会改变正在查看的历史 ID。

回答头部显示 `当前位置 / 总数`、上一条、下一条和按需出现的“回到最新”。浏览历史时新答案到达，只增加 `newAnswerAvailable` 提示，不强制跳页。每页使用该答案生成时保存的资料来源和选择 revision，不能拿当前资料替换历史依据。

### 5. 输入区改为单行紧凑双按钮组件

`LiveActionComposer` 收敛为 `CompactQuestionBar`：一个可自动增高但有最大高度的输入框，右侧两个次紧凑按钮“回答问题”和“截图回答”。桌面使用单行，手机可换成输入框一行、按钮并排一行。待确认问题卡片、标题和大面积计费说明不再放入该组件。

“回答问题”只处理手动输入；自动面试官问题由确认事件直接触发。“截图回答”打开现有截图流程且不清空手动草稿。按钮保持清晰文字和至少 44px 手机触控尺寸，但桌面视觉不使用大卡片。

### 6. 实时页隐藏点数，服务端计费语义不变

从自动问题、手动输入和截图按钮中移除“5 点 / 15 点 / 会员不限次”等文案，截图 Sheet 也不以点数作为主操作标签。积分页继续显示当前服务端目录费率、余额和明细；服务端仍在创建任务前原子校验权益并使用 usage ID 幂等预留。

余额不足时提交失败，实时页显示简短错误与“前往积分页”，不会进入生成状态。该决定优先减少现场认知负担，但会与此前“每次操作前展示费率”的活跃规范冲突，因此 apply 时必须明确修订对应规范和测试，而不能只改 UI。

### 7. 设备与会话状态进入顶部紧凑栏

顶部继续展示收音/重连/权限状态以及暂停、恢复和结束。详细设备列表仍在设备页面，实时页不再显示永久设备卡片。错误状态使用可执行提示，不依赖右栏。

### 8. 手机保持相同顺序且不提供资料抽屉

手机顺序固定为实时对话、答案、紧凑输入栏，不提供资料或历史抽屉。历史通过答案翻页完成。输入栏使用 `visualViewport` 或 CSS 动态视口单位避开软键盘，并继续使用 safe-area 留白。

## Risks / Trade-offs

- [没有右侧历史列表后无法一眼扫描所有问题] → 显示当前位置、总数、清晰翻页和“回到最新”，后续依据测试决定是否增加轻量目录弹层。
- [实时对话占用纵向空间] → 限制默认高度并允许内部滚动，答案仍获得更大的主要区域。
- [隐藏点数降低单次消费透明度] → 积分页保留明确费率与流水，余额不足不启动任务；该取舍需在用户测试中复核。
- [浏览历史时错过新答案] → 保持历史页并显示“有新答案”，不强制跳转。
- [拖动导致流式回答频繁重排] → 只更新 CSS 分栏比例，不卸载回答、对话或输入组件，并支持减少动态效果。
- [转录自动滚动打断阅读] → 仅在用户靠近底部时跟随最新内容。

## Migration Plan

1. 先补充工作台视图状态和答案 ID 翻页工具，保持旧 UI 可运行。
2. 新建 `ConversationMonitor`，迁移双角色转录和不清晰问题确认，不提供角色纠正。
3. 新建 `AnswerWorkspace`，迁移当前答案与历史翻页并验证新答案到达行为。
4. 将 `LiveActionComposer` 收敛为紧凑双按钮输入栏，复用已有手动与截图命令。
5. 移除资料栏，将设备/会话状态保留在顶部，并实现可访问分隔条与比例记忆。
6. 完成平板、手机、软键盘、安全区与可访问性测试。
7. 同步修订活跃实时工作台、主操作区、说话人确认和计费披露规范。

若新布局出现严重回归，可暂时恢复右栏容器用于诊断，但不得把手动回答、截图或历史翻页重新变成只能从右栏访问。

## Open Questions

- 实时对话区默认展示最近多少条片段，需要通过原型测试在信息完整与纵向空间之间确定。
- 历史答案未来是否需要问题目录弹层，本变更先验证上一条/下一条翻页。

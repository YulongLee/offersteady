## Why

当前实时面试页同时展示资料、回答、历史、转录、设备和多组操作，三栏布局压缩了真正需要阅读的回答空间，也让用户在现场难以判断注意力应该放在哪里。最终使用页需要围绕“看清双方正在说什么”和“快速查看面试官问题的答案”两件事重新收敛。

## What Changes

- 移除左侧面试资料栏和现场调整资料入口；回答仍展示生成时实际使用的资料来源。
- 移除现有右侧历史、实时转录和设备混合栏，不再为次要信息永久占用横向空间。
- 桌面将主区域简化为可拖动的左右两栏：左侧显示面试官与面试者的实时对话，右侧显示当前回答；窄屏降级为上下单列。
- 实时对话按角色、时间、最终/临时状态展示；回答系统只由已确认的面试官问题、用户手动提交的问题或截图题触发，面试者自己的回答不得自动生成新答案。
- 将历史回答整合到答案块内部，通过上一条、下一条、当前位置和“回到最新”翻页，不再依赖右侧历史列表。
- 将手动输入收敛为紧凑输入栏，在输入框旁放置“回答问题”和“截图回答”两个尺寸适中的按钮。
- 实时操作按钮和输入栏不展示每次使用的点数；积分余额、费率和明细继续在积分页面管理，余额不足或权益不可用时才在提交后提供明确恢复路径。
- 截图仍经过预览、识别、失败和重试流程；操作区变小不改变幂等扣费、失败不结算和隐私边界。
- 手机和平板保持相同的两块信息结构且不显示资料抽屉，翻页和两个操作按钮不得被底部控件或软键盘遮挡。

## Capabilities

### New Capabilities

- `focused-live-interview-workspace`: 定义无资料侧栏的可调整对话/回答工作台、答案与内嵌历史翻页、紧凑双操作输入栏，以及面试官问题优先触发规则。

### Modified Capabilities

无。相关实时工作台、说话人检测、主操作区和计费披露规范仍位于未归档变更中；本变更在实现前对它们进行统一收口和冲突修订。

## Impact

- `apps/web` 的 `LivePage`、`LiveActionComposer`、`SpeakerPanel`、历史展示和响应式 CSS 需要重构。
- 前端实时状态需要把对话时间线与回答分页状态分开，并在查看历史时继续接收最新转录和答案。
- 当前问题触发必须继续使用稳定问题 ID 和可信音频来源，防止面试者语音、重复转录或翻页操作产生重复回答和扣费。
- 计费服务端规则、余额校验和流水保持不变，但实时页不再承担逐按钮费率展示；相关活跃 Spec 与测试需要同步调整。
- 对话转录、问题、截图和回答均属于敏感数据；页面精简不得增加日志、通知或未授权设备中的内容暴露。

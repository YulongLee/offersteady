## Why

当前首页强调“回答依据”和产品边界，表达准确但不够直接，用户难以快速感受到拿 Offer、灵活付费和免费试用的价值。登录、支付、图片资产和使用说明也仍带有原型或人工流程痕迹，需要升级为可转化、可运营的正式产品入口。

## What Changes

- 将首页主标题调整为类似“AI 面试助手，助你更从容地冲刺 Offer”，副标题直接说明实时理解问题、结合个人资料生成回答思路、支持截图题和跨设备使用。
- 首页新增性价比模块，突出新用户 200 点免费额度、按次积分与按天会员两种方式，用户可以根据面试节奏选择，不暗示保证拿到 Offer。
- 将“进入产品原型”等内部阶段文案全部替换为“免费使用”“立即免费体验”等用户语言，并明确“新用户赠 200 点，无需先付费”。
- 当且仅当 Windows x64 版本具备已签名下载包、基础收音/连接能力和验证记录时，首页和下载中心统一显示“Windows 已支持”；不再使用“规划中”或“预览版”文案。
- 登录页点击微信登录后打开微信官方授权窗口或二维码弹层，成功回调后创建或绑定 OfferSteady 账号；本地原型登录只在开发环境显示。
- 新增统一图片资产目录和资产清单，管理面试稳 Logo/图标、微信与支付宝渠道图标、客服微信二维码、客服群二维码及其版本、用途和替换状态。
- 微信/支付宝支付二维码改为每笔订单由官方支付接口动态生成，不把固定收款码、交易单号或付款截图作为支付确认依据。
- **BREAKING**：移除购买弹层中的交易单号、付款截图上传和人工核验流程；支付状态只由微信支付/支付宝官方通知或服务端主动查单确认。
- 新增独立“使用说明”导航和 `/app/guide` 页面，覆盖首次使用、资料准备、设备连接、面试现场、截图题、积分会员、隐私、安全与故障处理，并允许业务页面跳转到对应章节。

## Capabilities

### New Capabilities

- `conversion-focused-public-entry`: 面向求职结果、免费试用和灵活付费的首页表达、CTA 与可验证产品证据。
- `verified-windows-product-support`: Windows 已支持声明、最低能力门槛、发布验证和跨页面一致性。
- `wechat-account-entry`: 微信官方授权弹层、登录注册、账号绑定、回调恢复和开发模式回退。
- `official-payment-checkout`: 微信/支付宝官方下单、动态支付二维码或跳转、回调查单、自动发放权益与退款。
- `managed-product-assets`: 品牌、支付渠道、客服和客服群图片资产的目录、清单、完整性与运营替换规则。
- `in-app-user-guide`: 独立使用说明页面、章节导航、搜索、上下文帮助和版本化内容。

## Impact

- `apps/web`：公开首页、登录页、应用导航、收费结算、设备下载和新增 `/app/guide`。
- `apps/api`：微信授权、微信支付/支付宝支付下单、通知验签、主动查单、退款和权益激活。
- `apps/desktop`：Windows x64 发布验证、基础能力门槛和支持状态报告。
- 资产结构新增 `apps/web/public/assets/{brand,payments,support}` 以及受控资产清单；生产客服二维码可由受控对象存储/CDN 覆盖，不提交私人收款凭证或密钥。
- 内容结构新增版本化中文使用说明，可从 Markdown 或结构化内容安全渲染。
- 需要微信开放平台、微信支付商户、支付宝开放平台、Windows 代码签名证书和公网 HTTPS 回调域名；密钥与证书不得进入客户端或代码仓库。
- 微信身份、订单和退款数据属于敏感数据，必须加密传输、限制访问、日志脱敏并提供明确隐私说明。
